# bbabam (빠밤!) Redesign - 템플릿 기반 젠더리빌 카드

## 개요

미니게임 기반 젠더리빌 앱을 **템플릿 기반 카드** 형태로 전환한다. 디자이너 없이도 관리 가능한 카드 템플릿 시스템으로, 사용자(부모)가 템플릿을 선택하고 정보를 입력하면 공유 가능한 젠더리빌 카드가 생성된다.

향후 확장: 미니게임 젠더리빌, 단순 임밍아웃 섹션 추가 예정.

## 디자인 방향

- **색상:** 핑크 + 블루 기반 파스텔 톤
- **폰트:** Jua (이미 설치됨) - 귀엽고 둥근 느낌의 한글 폰트
- **분위기:** 아기 친화적, 따뜻하고 귀여운 톤
- **레이아웃:** 모바일 퍼스트, max-width 480px, PC에서 가운데 정렬
- **헤더:** 뒤로가기(← ) + 로고(빠밤!) + 햄버거 메뉴(☰)
  - 랜딩: 로고 + 햄버거만 (뒤로가기 없음)
  - 카드 공유 페이지: 로고만 (받는 사람용 최소 UI)

## 사용자 흐름

### 카드 만드는 사람 (부모)

1. 랜딩 페이지 진입 -> "카드 만들기" CTA
2. **Step 1 - 템플릿 선택:** 그리드로 나열, 미리보기 이미지 + 인터랙션 유형 라벨
3. **Step 2 - 정보 입력:**
   - 아기 별명 (필수)
   - 성별 선택: 아들/딸 (필수)
   - 받는 사람 설정 (필수): 직접 입력 또는 "받는 사람이 적게 하기" 토글
   - 초음파 사진 업로드 (선택)
   - OG 이미지 모드: 기본 / 페이크 서프라이즈 (필수)
4. **Step 3 - 미리보기:** 받는 사람 시점에서 카드 체험 (인터랙션 포함)
5. **Step 4 - 완료:** 공유 링크 생성 + 복사 버튼 + 카카오톡 공유

비로그인 시: 카드 생성 가능, 로컬스토리지에 카드 ID 저장. 로그인하면 계정에 연결.

### 카드 받는 사람

1. 공유 링크 클릭
2. 페이크 서프라이즈 모드: 엉뚱한 화면(기프티콘 등) 먼저 표시 -> 탭하면 카드로 전환
3. 받는 사람 이름 입력 (해당 옵션 선택 시)
4. 카드 인터랙션 수행 (스크래치, 카드 뒤집기 등 템플릿별 고유 방식)
5. 성별 공개 + 축하 애니메이션 (컨페티 등)
6. 재방문 시: 매번 인터랙션부터 다시 시작

결과 화면에 "나도 카드 만들기" CTA 포함 (서비스 바이럴).

## 페이지 구조

| 라우트 | 설명 | 헤더 구성 |
|---|---|---|
| `/[locale]/` | 랜딩 - 서비스 소개 + "카드 만들기" CTA | 로고 + 햄버거 |
| `/[locale]/create` | 카드 생성 스텝 위자드 | 뒤로 + 로고 + 햄버거 |
| `/[locale]/c/[slug]` | 카드 공유 페이지 (받는 사람용) | 로고만 |
| `/[locale]/dashboard` | 내 카드 목록 + 링크 복사 + 삭제 | 뒤로 + 로고 + 햄버거 |
| `/[locale]/login` | 소셜 로그인 | 뒤로 + 로고 + 햄버거 |

### 햄버거 메뉴 항목

- 홈
- 카드 만들기
- 내 카드 (로그인 시)
- 로그인 / 로그아웃
- 언어 변경 (한국어/English)

## 카드 템플릿 시스템

- 템플릿은 코드에서 상수 + React 컴포넌트로 관리 (DB 아님)
- 각 템플릿은 고유한 인터랙션 방식을 가짐 (스크래치, 카드 뒤집기, 봉투 열기 등)
- MVP 템플릿 수와 종류는 구현 단계에서 결정
- 템플릿 구성 요소: id, name, interactionType, thumbnail, 카드 컴포넌트, 결과 컴포넌트

## OG 이미지

동적 OG 이미지 생성 (`/api/og/[slug]`):
- **기본 모드:** "OO의 성별은 무엇일까요?" 스타일의 미리보기
- **페이크 서프라이즈 모드:** 기프티콘 등 엉뚱한 이미지로 위장, 열어보면 젠더리빌 카드

카드 설정(ogMode)에 따라 서버에서 다른 이미지를 동적 생성.

## 데이터 모델

### users (기존 유지)

| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | PK |
| email | string | 이메일 |
| provider | string | 소셜 로그인 제공자 |
| createdAt | timestamp | 생성일 |

### cards

| 필드 | 타입 | 설명 |
|---|---|---|
| id | uuid | PK |
| slug | string | 공유용 고유 키 (unique) |
| userId | uuid (nullable) | 생성자 (비로그인 시 null) |
| templateId | string | 선택한 템플릿 ID |
| babyNickname | string | 아기 별명 |
| gender | enum (boy/girl) | 성별 |
| recipientMode | enum (preset/input) | 받는 사람 설정 방식 |
| recipientName | string (nullable) | preset 모드일 때 받는 사람 이름 |
| ogMode | enum (default/fake-surprise) | OG 이미지 모드 |
| ultrasoundImageUrl | string (nullable) | 초음파 사진 URL |
| language | enum (ko/en) | 카드 언어 |
| createdAt | timestamp | 생성일 |

## 기술 스택 (기존 유지)

- **프레임워크:** Next.js 16 (App Router)
- **언어:** TypeScript
- **스타일링:** Tailwind CSS 4 + Framer Motion
- **폰트:** Jua (Google Fonts / 로컬)
- **DB:** Drizzle ORM + Neon PostgreSQL
- **인증:** next-auth v5 (소셜 로그인)
- **i18n:** next-intl (한국어/English)
- **파일 저장:** Vercel Blob (초음파 사진)
- **OG 이미지:** Next.js 동적 OG 생성
- **배포:** Vercel

## API 라우트

| 라우트 | 메서드 | 설명 |
|---|---|---|
| `/api/cards` | POST | 카드 생성 |
| `/api/cards` | GET | 내 카드 목록 (인증 필요) |
| `/api/cards/[id]` | DELETE | 카드 삭제 (인증 필요, id는 uuid) |
| `/api/cards/by-slug/[slug]` | GET | 카드 정보 조회 (공유 페이지용, slug는 공유 키) |
| `/api/upload` | POST | 초음파 사진 업로드 |
| `/api/og/[slug]` | GET | 동적 OG 이미지 |

## 비로그인 카드 관리

- 카드 생성 시 로그인 불필요
- 생성된 카드 ID를 로컬스토리지에 저장
- 로그인 시 로컬스토리지의 카드 ID를 조회하여 해당 카드들의 userId를 현재 사용자로 업데이트
- 로컬스토리지 키: `bbabam_cards` (카드 ID 배열)
