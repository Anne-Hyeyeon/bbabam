# 빠밤! Design Principles

> Feature: `overhaul` · Version: v1 · 2026-04-17
> 관련 문서: `docs/00-pm/overhaul.prd.md`

## 1. 디자인 방향

**"Linear의 뼈대 + Duolingo의 온기"**

- 정리된 그리드와 깔끔한 타이포(Linear/Notion 계열)를 기본 뼈대로 하되
- 따뜻한 컬러 팔레트, 손그림풍 일러스트, 마스코트 캐릭터로 친근함을 더한다
- 의료·서비스 업계의 차갑고 기능적인 UI 대신, 임신·출산의 **기쁨과 설렘**을 담는다

### Anti-Pattern (피할 것)
- 파란색 + 흰색 위주의 "병원 앱" 톤
- 딱딱한 직각 카드, 강한 그림자, flat but cold
- 지나치게 컬러풀한 어린이집 톤 (우리 타깃은 예비 부모)
- 과한 그라데이션, 네온, Y2K
- 결혼식장 청첩장 톤 (지나친 금박/장식)

- **전체 스택**: `'Pretendard', 'Inter', system-ui, -apple-system, sans-serif`

## 4. 스페이싱 & 그리드

### 4.1 8pt 그리드

- 기본 단위: `4px`
- 자주 사용: `8 / 16 / 24 / 32 / 48 / 64`
- 페이지 좌우 여백(모바일): `20px`
- 페이지 좌우 여백(데스크톱): `24-48px` (`max-w-6xl mx-auto`)

### 4.2 모바일 퍼스트

- 모든 섹션은 iPhone 13 기준(390px)에서 먼저 검증
- 반응형 브레이크포인트: `sm 640 / md 768 / lg 1024 / xl 1280`
- 타깃 사용자가 모바일 비중 절대적 (예비 엄마 맘카페/인스타 유입)

## 6. 아이콘 & 일러스트

### 6.1 아이콘

- **라이브러리**: `lucide-react` (shadcn/ui 기본)
- Stroke width: 1.75 (기본 2에서 조금 얇게 — 따뜻한 인상)
- Size: 16 / 20 / 24 / 32
- 아이콘만 단독으로 쓰지 말고 가능한 레이블과 함께

### 6.2 일러스트

- **스타일**: **손그림풍** (확정)
  - 삐뚤빼뚤한 라인, 손칠한 듯한 색칠
  - 완벽한 베지에 곡선 X, 약간 불규칙한 선
- **소재**: 초음파, 아기발, 하트, 아기 볼, 풍선, 구름, 별
- **컬러**: 위 팔레트의 파스텔 악센트 사용
- **사용처**: 빈 화면(empty state), 섹션 헤더, 결과 카드 배경
- **과하지 않게**: 한 페이지에 일러스트 3개 초과 금지

### 6.3 일러스트 제작 방법 (확정: 무료 라이브러리 커스텀)

**확정**: **무료 라이브러리 커스텀** (빠른 MVP 우선)

- 1차 추천: **Storyset** (illustrations.co 계열) — hand-drawn 스타일 변형 제공, SVG 커스텀 가능
- 2차: **unDraw** — 단색 커스텀 쉬움, 톤 통일 유리
- 3차: **Open Peeps / Blush** — 인물 손그림풍, 조합 가능
- **톤 통일 전략**: SVG로 받아서 primary/accent 색을 디자인 토큰으로 **일괄 치환** → 전체 일관성 확보
- 톤이 안 맞는 리소스는 쓰지 않는다 (차라리 없이 가는 게 낫다)

## 7. 마스코트

### 7.1 역할

- Duolingo 올빼미·Mailchimp Freddie처럼 **조용한 친근감** 제공
- 랜딩·결과 카드·empty state·튜토리얼 가이드에 소량 배치
- **브랜드 아이덴티티 강화** — 맘카페 추천 글에서 "그 캐릭터 있는 거!" 로 회상

### 7.2 컨셉 (확정: 토끼 캐릭터)

**확정**: **토끼 캐릭터** (가칭 "빠토 / 밤토" — 네이밍은 별도 결정)

- 한국에서 임신·아기 이미지로 친숙 (토깽이)
- 따뜻하고 친근한 인상, 유아용 콘텐츠와 잘 어울림
- 긴 귀·둥근 몸·짧은 꼬리 — 실루엣 인지성 ↑
- 손그림풍 라인 + 파스텔 컬러로 표현

### 7.3 마스코트 가이드라인

- 기본 포즈 세트 (최소 5개): 인사 / 놀람(깜짝) / 축하(폭죽) / 잠자기 / 생각 중
- 성별 힌트 상태: 분홍 리본 (여아 힌트) / 파랑 리본 (남아 힌트) — 섹션 4 젠더리빌 전용
- 제작 방법: 무료 라이브러리 커스텀 (§ 6.3 A)

## 8. 모션

### 8.1 원칙

- **설렘을 보조하되 방해하지 않는다**
- 이미 설치된 `framer-motion` 활용
- 페이지 전환: 200-300ms ease-out 페이드·슬라이드
- 버튼 호버: scale 0.98~1.02, 150ms
- 카드 등장: stagger (섹션 내 카드들이 50ms 간격으로 fade-in)
- **화려한 애니메이션 금지**: 회전, 튕김, 강한 scale, 색상 깜빡임

### 8.2 특별 모션 (섹션 3·4 카드 전용)

- 젠더리빌/임밍아웃 카드는 **공개의 순간**이 핵심 → 별도 공을 들인 모션 허용 (스크래치, 플립, 풍선 터지기)
- 다만 "축하 폭죽" 류는 1회성으로만, 반복 재생 금지

## 9. 컴포넌트 라이브러리

### 9.1 베이스

- **shadcn/ui + Tailwind v4** (이미 설치됨)
- Radix 프리미티브 기반 → 접근성 확보됨
- 필요한 컴포넌트 순차 설치:
  - `button`, `input`, `card`, `dialog`, `tabs`, `badge`, `tooltip`, `skeleton`
  - 섹션별 필요 시 확장 (예: 섹션 1의 퀴즈 UI는 `radio-group` 필요)

### 9.2 커스텀 래핑

shadcn 기본을 그대로 쓰지 말고, 프로젝트 디자인 토큰으로 **한 겹 래핑**:

- `<Button>` → Coral primary / Warm secondary / Ghost variant
- `<Card>` → radius-lg + shadow-card + hover transition
- `<SectionHeader>` → h2 + 선택적 일러스트 슬롯

### 9.3 섹션별 공통 컴포넌트

- `<ResultCard>`: 6개 섹션 모두 "결과"가 공유 자원 → 단일 컴포넌트로 (카드 배경, 일러스트, 공유 버튼, 홈 버튼)
- `<ShareBar>`: 카카오·인스타·링크복사 통일
- `<MascotGuide>`: 마스코트 + 말풍선 (튜토리얼·empty state 공용)

## 10. 접근성

- 컬러 대비: WCAG AA 이상 (본문 4.5:1, 큰 텍스트 3:1)
- 모든 인터랙티브 요소에 포커스 링 (Coral 톤)
- 이미지·일러스트에 `alt` 필수
- 모션 설정에서 `prefers-reduced-motion` 존중 (강한 모션 off)
- 한국어 입력 IME 고려 (폰트 사이즈 16px 이상으로 자동 줌인 방지)

## 11. 톤 & 보이스 (카피라이팅)

- **친근한 반말 톤** 기본 ("짜잔!", "맞춰봐", "시작해보자")
- **병원 앱스러운 딱딱한 존댓말 X** ("테스트를 진행합니다" → "한번 해볼까?")
- **과도한 이모지 X** (한 문장에 이모지 1개 이내)
- 결과 문구에 **감성 한 스푼** ("우리 아기, 반짝이는 별 같아요")

## 12. 확정 결정 사항

- **마스코트**: 토끼 캐릭터 (§ 7.2)
- **일러스트 제작**: 무료 라이브러리 커스텀 (§ 6.3)
- **폰트 로딩**: `next/font` 사용 (셀프 호스팅, SSR 시 FOIT/FOUT 제거)
- **컬러 팔레트 & 타이포**: § 2, § 3 확정
