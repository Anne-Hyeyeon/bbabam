## 1. Executive Summary

### 1.1 4-Perspective Summary

| 관점 | 핵심 요약 |
|------|-----------|
| **Problem (문제)** | 한국 2030 예비 부모는 임신 확인 직후부터 출산까지 "재미있는 걸로 시간 때우기"(성별 맞추기 속설, 태명 고민, MBTI 테스트)와 "알리고 축하하기"(임밍아웃/성별 공개 카드) 두 가지 욕구를 모두 갖는다. 그러나 이들은 맘카페·블로그·개별 앱에 **파편화**되어 있어, 하나의 브랜드가 이 전체 journey를 커버하지 못한다. |
| **Solution (해결)** | **빠밤!은 임신/출산 관련 가볍고 재미있는 도구·콘텐츠를 한 사이트에 모은 포털**이다. 기존 "빠밤 카드"는 이 포털의 젠더리빌 섹션으로 흡수되며, 성별 맞추기·아기 유전자 예상·태명 생성기·부모 MBTI 등 **6개 섹션**이 각기 다른 유입/리텐션/공유 역할을 수행한다. **모든 섹션은 AI를 사용하지 않고**, 룩업 테이블·큐레이션 DB·룰 기반 매칭으로 결정론적 결과를 제공한다. |
| **Function UX Effect (기능/UX 효과)** | (1) SEO 친화 도구(성별 맞추기·유전자 예상·MBTI)로 **저비용 오가닉 유입**, (2) 카드 섹션으로 **viral 공유 loop**, (3) 태명/MBTI 결과 저장으로 **리텐션 후킹**, (4) 섹션 간 크로스링크로 평균 페이지뷰↑ → **AdSense 단가↑**. AI 런타임 비용이 0이라 AdSense RPM이 즉시 마진이 됨. |
| **Core Value (핵심 가치)** | "임신 확인부터 출산까지, 심심할 때마다 돌아오게 되는 한국형 예비부모 놀이터." |

### 1.2 왜 지금(Why Now)인가

- 2025년 한국 출생아 수 258,242명, **9년 만에 +6.56% 반등** — 2030 예비 부모 풀 회복 (출처: 행정안전부, Korea.net)
- 2030 세대의 임신 관련 정보 탐색은 **맘카페·인스타·단편 블로그에 파편화** → 통합 포털 기회
- banggooso.com 같은 **"가벼운 도구 포털" 모델**이 한국에서 긴 체류시간과 AdSense 수익을 입증
- 기존 단일 카드 제품만으로는 **재방문 동기가 약함** → 포털화로 LTV/AdSense 수익 동시 확보
- AdSense 관점에서 **결정론적(deterministic) 콘텐츠 기반 포털**은 정책 리스크·운영 비용이 모두 낮아, 부트스트랩 단계에서 유리

**AI를 빼기로 한 배경**:
- AI 런타임 비용 없음 → AdSense RPM이 곧바로 마진.
- AdSense 의료/건강 주장 정책 위반 리스크 대폭 감소 (LLM 환각 제거).
- 결과가 결정론적이라 QA/테스트·캐싱·공유용 OG 생성이 단순.
- 개발 scope가 명확해져 6개 섹션 순차 런치가 현실적.

## 2. Current State Assessment (현재 상태 진단)

### 2.1 왜 피벗하는가 (v1 → v2 전환 맥락)

v1 PRD(단일 카드 제품)의 근본 한계:

| 한계 | v2에서의 해결 |
|------|---------------|
| **1회성 유스케이스**: 성별 공개는 임신 중 1회 이벤트 → LTV 낮음 | 6개 섹션 간 cross-journey로 임신 초기 ~ 출산까지 체류 |
| **재방문 유인 없음**: 카드 1장 만들고 이탈 → 북마크할 이유 없음 | 태명/MBTI 저장, 성별맞추기 속설 도감 등 **돌아올 이유** |
| **수익화 경로 불투명**: 프리미엄 템플릿 WTP 검증 안 됨 | **AdSense 중심의 체류시간·페이지뷰 비즈니스**로 단순화 |
| **포지셔닝 모호**: "카드" 제품은 바른손·보자기의 복제 리스크 | "임신/출산 도구 포털"이라는 **새 카테고리**로 차별화 |

### 2.2 현재 자산 (v1에서 v2로 이관되는 코드/디자인)

- **기술 스택**: Next.js 16 + React 19 + Drizzle ORM + NextAuth v5 (Kakao/Google), `@vercel/blob`, `framer-motion`, next-intl (`ko`/`en`)
- **재활용되는 코드**:
  - `src/components/create/*` 및 `src/components/viewer/*` → **젠더리빌 카드 섹션**으로 편입
  - `src/components/templates/*` (scratch/flip/envelope) → 젠더리빌의 "단순형" 카드 변형
  - 동적 OG 이미지, fake-surprise 모드, 공유 플로우 → 전 섹션의 **공유 인프라**로 승격
- **디자인 자산**: 파스텔 톤 시스템(`#F472B6`, `#60A5FA`, `#FFF7ED`, `#6EE7B7`), Jua 폰트 — 포털 전 섹션 브랜드 톤으로 유지
- **인증/DB**: NextAuth + Drizzle → 포털 공통 사용자 계정 (섹션별 결과 저장)

### 2.3 구현되지 않은 것 (v3에서 새로 필요)

- 포털 랜딩(6 섹션 허브)
- 섹션 라우팅 IA (`/gender-predict`, `/baby-genetics`, `/immingout`, `/gender-reveal`, `/taemyeong`, `/parent-mbti`)
- **유전자 룩업 테이블 데이터 구축** (섹션 2) — 눈동자 색/머리카락/혈액형/키 통계 등 우성·열성 조합 Punnett square 데이터
- **태명 큐레이션 DB + 룰 매칭 엔진** (섹션 5) — 수동 큐레이션한 태명 사전(수백~수천 개) + 필터(계절/이니셜/느낌)
- AdSense 슬롯 배치 및 콘텐츠 정책 대응
- 섹션 간 크로스링크 엔진 (예: MBTI 결과 → "우리 부부에 어울리는 태명 만들기" CTA)

### 4.2 포털형 Beachhead — 단일 엔트리가 아닌 **다중 엔트리**

v1은 "임신 16~24주 예비 엄마"라는 단일 진입 구간을 가정했으나, **포털은 다양한 엔트리 포인트로 유입되는 것이 설계 목표**.

| 임신 단계 | 주요 검색 의도 | 걸리는 섹션 (1st touch) | Cross-link 유도 |

### 6.4 크로스링크 엔진 설계 원칙

- **섹션별 Outbound 3슬롯**: 각 섹션 결과 페이지 하단 3개 CTA (관련 섹션 2개 + 저장 1개)

## 7. Monetization

### 7.1 초기 전략: 무료 + AdSense (선콘텐츠 · 후승인)

**결정 (확정)**: 모든 섹션 무료 공개, **Google AdSense**로 단일 수익화. **선콘텐츠 · 후승인** 경로 확정.

**근거**:
- 포털은 **체류시간·페이지뷰 비즈니스** — AdSense와 구조적 궁합
- 프리미엄 템플릿 WTP는 검증 안 됨, 유료화는 traction 이후
- 포털 성격상 **진입 마찰 제로**가 성장 엔진
- 非AI 구성으로 **런타임 비용이 호스팅/DB 수준**이라, AdSense RPM이 바로 마진

**선콘텐츠 · 후승인 운영 원칙**:
1. Phase 1 런치 시 AdSense 신청 **보류** — 광고 슬롯 코드는 배치해두고 비노출
2. 섹션 1(성별 맞추기 도구 3종, 각 1,000단어+ 속설 해설) · 섹션 6(MBTI 16유형 상세 페이지, 각 1,000단어+)의 SEO 콘텐츠가 충분히 쌓인 시점에 AdSense 신청
3. 승인 후 광고 슬롯 활성화 (섹션별 배치 정책은 7.2 참조)

### 7.4 AdSense 정책 리스크 (반드시 대응)

- **의료 주장 금지**: 섹션 1(성별 맞추기 속설)·섹션 2(유전자 예상)는 "재미로 보는 속설" / "교과서 수준 일반 확률, 실제 의료 진단 아님" 면책 푸터 필수. **AI 없이도 콘텐츠 자체가 의료 뉘앙스로 오인될 수 있음** — 원인이 AI가 아니라 **콘텐츠 자체**이므로 정책 대응 여전히 필요.
- **아동/임신 콘텐츠 정책**: AdSense 정책상 어린이 타겟 아님 명시 필요 (타겟은 예비 부모)
- **확정적 유전 예측 금지**: 섹션 2 결과 페이지는 반드시 "확률적 참고용, 실제 자녀 표현형은 다를 수 있음" 문구 고정
- **선정성 없음**: 모든 섹션 가족 친화 유지
- **큐레이션 콘텐츠 저작권**: 섹션 5 태명 DB·섹션 1 속설 해설은 **직접 큐레이션/작성**하여 2차 저작권 리스크 제거

| **Strengths** | 브랜드 인지, 템플릿 다양, 결혼·돌 시장 강력 점유 |
| **Weaknesses** | **카드 단일 제품**, 인터랙션 없음, 임신 초기~중기 도구 부재, 성별 공개 템플릿 드묾 |
| **Pricing** | 카드 건당 2~5만 원 유료 포함 |
| **빠밤! vs** | 우리는 **포털** — 카드(섹션 3·4)는 그중 하나, 그들은 카드가 전부 |
| **How to Win** | 카드 앞단의 성별 맞추기·태명·MBTI로 **유저가 카드 만들기 훨씬 전에 우리를 먼저 만남** → funnel 상단 선점 |

### 8.3 맘카페 / 개별 블로그 (파편화된 UGC)

| 항목 | 내용 |

## 9. Growth Loops (섹션별 역할 분리)

### 9.1 Loop #1 — SEO 유입 Loop (섹션 1, 6)

```
구글/네이버: "아들 딸 구별법", "임신 입덧 아들"
 → 섹션 1 (성별 맞추기) 페이지 진입
 → 테스트 완료 → 결과 페이지
 → Cross-link: 섹션 5(태명) / 섹션 6(MBTI)
 → 2개 이상 완료 → /saved 북마크 유도
```

**측정 지표**: 오가닉 세션, 페이지/세션, 체류시간, Cross-link CTR

**K-factor 기여**: 낮음 (공유 의도 약함)
**SEO 기여**: **매우 높음** — 저비용 상시 유입

### 9.2 Loop #2 — Viral 공유 Loop (섹션 3, 4)

```
임신 중기 민지: 섹션 4(젠더리빌 카드) 생성
 → 카톡으로 10명에게 공유
 → viewer 10명이 카드 열기 (게임 클리어 or 초음파 reveal)
 → 하단 "나도 만들기" + "빠밤 다른 도구 보기(MBTI/태명)" CTA
 → viewer 중 N명이 MBTI/태명으로 유입 → 섹션 1·5·6에 재유입
 → 원래 카드 만드는 사람도 증가 (2차 바이럴)
```

**측정 지표**: K-factor, 카드당 viewer 수, viewer→creator 전환율, viewer→섹션 N 이동률

**K-factor 기여**: **핵심** (목표 K ≥ 1.2)
**SEO 기여**: 중간 (카드 결과 페이지는 익명 slug)

### 9.3 Loop #3 — 결과 공유 Loop (섹션 2, 6)

```
섹션 2(아기 유전자 예상) or 섹션 6(MBTI) 완료
 → 결과가 "재미있고 공유하고 싶은" 형태 (이미지 카드: 확률 표 / MBTI 결과 카드)
 → 인스타 스토리 / 카카오 스토리 공유
 → 해시태그 #빠밤_임신MBTI #빠밤_아기유전자
 → 친구가 보고 본인도 해보기
```

**측정 지표**: 결과 페이지 공유 CTR, UGC 해시태그 언급 수, 해시태그 → 유입 세션

**K-factor 기여**: **중간** (viral 의도 중간)
**SEO 기여**: 낮음 (해시태그 기반)

### 9.4 Loop #4 — 리텐션 Loop (섹션 5, 저장함)

```
섹션 5(태명): 3~5개 제안 저장 → 부부가 며칠 걸쳐 결정
 → 2~3일 후 재방문하여 저장함에서 재검토
 → 재방문 시 다른 섹션 배너 노출 → 섹션 1·4·6 재유입
 → /saved 에 태명·MBTI·카드가 쌓임 → 북마크 대상이 됨
```

**측정 지표**: D7 재방문율, /saved 페이지 조회 수, 저장 후 재방문 간격

**K-factor 기여**: 낮음
**SEO 기여**: 낮음 — 대신 **LTV·AdSense impression 극대화**

- **완화**:

  - 모든 섹션 푸터 "본 서비스는 재미를 위한 엔터테인먼트 도구입니다. 의료 진단·상담을 대체하지 않습니다" 고정

## 11. User Stories + Job Stories

### 11.1 Primary JTBD (포털 관점 6-Part)

> **When** I find out I'm pregnant (especially early trimester when I can't tell anyone yet),
> **I want to** have one warm, playful Korean site with multiple small tools and cards for pregnancy journey,
> **so I can** fill the anxious waiting time with joy and create shareable memories,
> **while** each tool takes under 5 minutes and everything is free,
> **without** hunting across scattered mom-cafes, blogs, and foreign sites,
> **I feel** excited, less alone, and connected to this journey.

### 11.2 섹션별 대표 User Stories / Job Stories

**섹션 1 — 성별 맞추기**
- US-1.1 (P0): 예비 엄마로서, 입덧 증상을 체크하고 "아들일 확률"을 재미로 보고 싶다. 이 시기 가장 큰 궁금증이라서.
- US-1.2 (P1): 중국황실달력 도구를 한국 나이로 자동 변환해주면 좋겠다. 계산하기 귀찮아서.
- JS-1.1: 임신 8주에 아직 성별을 모를 때, **"아들 딸 구별법 속설"을 한 자리에서 다 해보고** 싶다. 그래야 불안을 재미로 치환할 수 있다.

**섹션 2 — 아기 유전자 예상하기 (非AI)**
- US-2.1 (P0): 예비 부모로서, 엄마·아빠의 눈동자 색/머리카락/혈액형/키 등을 선택하면 **중·고등 생물 교과서 수준의 Punnett square 확률표**를 받아 "아이 눈은 갈색일 확률 75%" 같은 참고 정보를 재미로 보고 싶다.
- US-2.2 (P0): 결과는 **확정이 아닌 확률·범위 기반**으로 표시되어, 의료 오해 없이 "엔터 용도"임이 분명해야 한다.
- US-2.3 (P1): 형질별(눈/머리/혈액형/키) 탭을 오가며 여러 조합을 해볼 수 있어야 한다.
- US-2.4 (P1): 결과 카드를 인스타/카톡에 공유할 수 있어야 한다.
- JS-2.1: 임신 계획 중 부부끼리 **"우리 애기는 누굴 닮을까"** 대화할 때, **검색해서 여기저기 흩어진 유전 확률 정보**를 한 페이지에서 재미로 확인하고 싶다.

**섹션 3 — 임밍아웃 카드**
- US-3.1 (P0): 임신 12주 차에 가족/친한 친구에게 **카톡으로 임신 사실을 특별하게 알리고** 싶다. 단순 문자는 아쉬워서.
- US-3.2 (P1): 파트너 이름도 같이 들어가는 카드가 필요하다.
- JS-3.1: 안정기 진입 직후, **"임신했어요" 공지를 한 번에 예쁘게** 돌리고 싶을 때, 10초에 카드를 만들고 싶다.

**섹션 4 — 젠더리빌 카드 (기존 빠밤 카드 흡수)**
- US-4.1 (P0): 예비 엄마로서, **받는 사람이 게임을 클리어해야 성별이 공개되는 카드**를 만들고 싶다. "짜잔!" 순간이 기억에 남아야 해서.
- US-4.2 (P0): 단순 초음파 사진만 올려도 되는 간단 버전도 있어야 한다.
- US-4.3 (P1): 카드 하단에 "빠밤의 다른 도구들" CTA가 있어 친구가 MBTI도 해볼 수 있으면 좋겠다.
- JS-4.1: 성별 확인 후 친구들에게 카톡으로 알릴 때, **게임 클리어 → 성별 reveal** 경험이 사진 한 장보다 훨씬 특별하다.

**섹션 5 — 태명 생성기 (非AI, 큐레이션 DB + 룰 매칭)**
- US-5.1 (P0): 부부로서, 계절/부부 이니셜/원하는 느낌(귀여움·고급·의미) 같은 몇 가지 필터를 선택하면 **큐레이션된 태명 DB에서 매칭 결과 5~10개**를 의미·한자·발음과 함께 받아보고 싶다.
- US-5.2 (P0): 마음에 드는 태명을 저장하고, 며칠 뒤 다시 와서 고를 수 있어야 한다.
- US-5.3 (P1): 필터 조합을 바꿔가며 여러 번 탐색할 수 있어야 한다 (rate limit 없음, 결정론적이라 비용 이슈 없음).
- US-5.4 (P1): 선택한 태명을 예쁜 카드로 만들어 가족에게 공유하고 싶다.
- JS-5.1: 임신 확인 직후 부부가 태명을 짓기 시작할 때, **막막함을 덜어줄 시작점**이 필요하다. 맘카페 검색 대신 **필터 몇 번으로 후보군**을 얻고 싶다.

**섹션 6 — 예비부모 MBTI**
- US-6.1 (P0): 예비 부모로서, 12~16문항으로 **내 육아 스타일 MBTI 유형**을 받고, 공유 가능한 결과 카드를 얻고 싶다.
- US-6.2 (P0): 16유형별 상세 페이지가 SEO로 검색되어야 한다 ("ENFP 엄마 유형" 등).
- US-6.3 (P1): 파트너와 유형을 비교할 수 있는 궁합 페이지가 있으면 좋겠다.
- JS-6.1: 심심할 때 친구들이 하는 MBTI 테스트처럼, **예비 부모용 MBTI**를 하면서 내 육아 성향을 미리 알고 싶다.

### 11.3 INVEST 체크 (Sample: US-4.1)

- **I**ndependent: ✅ 기존 카드 코드 재활용
- **N**egotiable: ✅ 게임 타입 수(3~6종) 협상 가능
- **V**aluable: ✅ viral 핵심 자산
- **E**stimable: ✅ 기존 코드 기반
- **S**mall: ✅ 게임 1종 1~2 스프린트
- **T**estable: ✅ "게임 클리어 → reveal" 성공률 측정

## 12. Test Scenarios (섹션별 L1/L2/L3)

### 12.1 L1 — API / Unit

**섹션 1 (성별 맞추기)**
- 중국황실달력 API가 입력(한국 나이, 임신 월)에 대해 결정론적 결과 반환
- 입덧/입맛 테스트 점수 계산 로직 단위 테스트

**섹션 2 (아기 유전자 예상) — 룩업 테이블**
- `getPunnettProbability(trait, parentA, parentB)` 단위 테스트: 형질별 우성/열성 조합에 대해 기대 확률 반환 (예: 갈색 눈 × 파란 눈 → 갈색 75%, 파란 25%)
- 모든 지원 형질(눈동자/머리카락/곱슬/혈액형/키/보조개)에 대한 테이블 커버리지 테스트
- 입력 누락 시 기본값 처리 및 에러 메시지
- 결과 페이지 메타 태그에 "참고용, 실제 결과와 다를 수 있음" 문구 포함 확인

**섹션 5 (태명) — 큐레이션 DB + 룰 매칭**
- `matchTaemyeong(filters)` 단위 테스트: 필터(계절/이니셜/느낌) 조합에 대해 DB 쿼리가 5~10개 결과 반환
- 필터 조합별 최소 결과 개수 보장 (예: 가장 희귀한 조합도 최소 3개)
- 중복 의미/발음 제거 로직
- DB 스냅샷 테스트 (큐레이션 개수·태그 커버리지 회귀 방지)
- 결과 캐싱 (같은 필터 조합은 빠른 재조회)

**섹션 3·4 (카드)**
- 기존 `POST /api/cards` 테스트 승계 + 섹션 타입 파라미터 추가
- OG 이미지가 섹션별로 다른 테마 반환

**섹션 6 (MBTI)**
- 16유형 판별 로직 단위 테스트 (답변 조합 → 유형)
- 결과 페이지 slug 생성 및 유형별 메타태그

### 12.2 L2 — UI 액션

- 섹션 1: 3종 도구 간 네비게이션 + 결과 페이지 cross-link CTA 클릭
- 섹션 2: 형질 탭 전환(눈/머리/혈액형/키) → 부모 값 선택 → 즉시 확률 결과 표시 → 공유 카드 생성
- 섹션 3·4: 기존 wizard UI 작동 (템플릿 선택 → 편집 → 공유)
- 섹션 5: 필터 선택 → 결과 리스트 → 카드 저장/삭제 → 필터 변경 후 재탐색
- 섹션 6: 문항 진행 상태 저장 (이탈 후 복귀 시 재개)

### 12.3 L3 — E2E 시나리오

**시나리오 A — 임신 초기 사용자의 첫 방문 플로우**
1. 구글 검색으로 섹션 1 랜딩
2. 입덧 테스트 완료 → 결과 페이지 → Cross-link 섹션 6 클릭
3. MBTI 진단 완료 → 결과 저장 → 저장함 노출
4. 재방문 시 저장함 → 섹션 5 추천 배너 노출 → 태명 생성

**시나리오 B — Viral Loop (섹션 4)**
1. 예비 엄마 A 섹션 4에서 게임형 카드 생성
2. 카톡 공유
3. Viewer B 링크 클릭 → 게임 클리어 → 성별 reveal
4. Viewer B에게 "나도 만들기" + "섹션 6 MBTI 해보기" CTA
5. Viewer B가 섹션 6 유입 → 전환

**시나리오 C — 선콘텐츠 · 후승인 AdSense 플로우 검증**
1. Phase 1 런치 상태(AdSense 미승인)에서 광고 슬롯이 **보이지 않음** 확인
2. 승인 후 환경 변수/플래그 활성화 → 섹션별 슬롯 노출 확인
3. 섹션 2(유전자) 결과 페이지, 섹션 1(속설) 결과 페이지에 광고가 **UX를 방해하지 않는 위치**에 노출
4. viewer 페이지(`/c/[slug]`)는 Non-personalized ads 설정 확인

**시나리오 D — AdSense 정책 검증 (콘텐츠 안전성)**
1. 섹션 2 결과 페이지에 "엔터테인먼트 목적, 의료 진단 아님" 면책 노출 확인
2. 섹션 1·2 카피에 "100% 확실", "반드시 아들" 등 **확정 서술이 없음** 확인
3. 모든 섹션 푸터 정책 링크 노출
4. Phase 1 콘텐츠 길이 검증 (섹션 1 도구 페이지 · 섹션 6 16유형 페이지가 각 1,000단어 이상)

## Appendix C — Market Sizing (포털 버전)

v1의 카드 단일 계산 대신, **포털형 체류시간·AdSense 모델**로 재산정.

### Top-down

- **TAM**: 한국 연간 임산부 30만 명 × 연간 세션 20회(임신 기간 분산) = **600만 세션/년**
  - AdSense RPM ₩2,000~5,000 가정 → **TAM ≈ ₩12억~30억/년 (광고 수익 기준)**
- **SAM**: 디지털 액티브 70% → **SAM ≈ ₩8억~21억/년**
- **SOM (3년차, 점유 5%)**: **SOM ≈ ₩4천만~1억/년**

### Bottom-up

- MAU 1만 × 세션당 평균 3 페이지뷰 × AdSense CPM ₩3,000 × 12개월 = **약 ₩1~1.5억/년 (3년차 목표)**

### v1 대비 변화

- v1 (카드 단일): 3년차 ₩1~2억/년 (프리미엄 템플릿 10% 전환 가정)
- v2 (포털 + AdSense): 3년차 ₩1~1.5억/년 (유료 전환 0%, 광고만)
- **결론**: 단기 수익은 유사하나, **유저 풀 / 브랜드 자산 / LTV 확장 여지는 v2가 압도적으로 큼**
